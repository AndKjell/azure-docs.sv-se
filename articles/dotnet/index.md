# <a name="azure-libraries-for-net"></a>Azure Libraries för .NET

Använd biblioteken i .NET SDK för Azure för att hantera och använda Azure-tjänster i dina program.

## <a name="packages"></a>Paket

> [!TIP]
> [Flytande bibliotek](https://azure.microsoft.com/blog/simpler-azure-management-libraries-for-net/) (slutar på **\*.Fluent**) förbättrar utvecklingen genom att tillhandahålla en objektorienterad API på högre nivå som är optimerad för läs- och skrivbarhet. De klargör vad som krävs för valfritt kontra icke-ändringsbart. Biblioteken kan köras sida vid sida med icke-flytande bibliotek, så använd flytande paket om du föredrar en sådan syntax. [Microsoft.Azure.Management.Fluent](https://www.nuget.org/packages/Microsoft.Azure.Management.Fluent) är ett samlat paket som innehåller alla flytande bibliotek.

## <a name="installation"></a>Installation

### <a name="visual-studio"></a>Visual Studio

Om du använder Visual Studio kan du använda **NuGet Package Manager-konsolen** för att importera paketet till projektet.

1. När du har Visual Studio öppet startar du konsolen genom att klicka på **Verktyg**, **NuGet Packager Manager** och sedan på **Package Manager Console** (Package Manager-konsol).  

    ![Package Manager-konsol](media/index/package-manager.png)

2. I konsolfönstret använder du cmdlet:en **Install-Package** för att ladda ned och installera NuGet-paketet.  För att exempelvis inkludera den senaste versionen av [Azure Resource Management Library](http://www.nuget.org/packages/Microsoft.Azure.Management.ResourceManager) för .NET:

    ```powershell
    Install-Package Microsoft.Azure.Management.ResourceManager -Pre 
    ``` 
    Om du vill använda en specifik version inkluderar du versionsnumret så här:

    ```powershell
    Install-Package Microsoft.Azure.Management.ResourceMananger -Version 1.4.0-preview
    ``` 

### <a name="other-editors"></a>Andra redigerare

Om du använder .NET Core med Visual Studio Code (eller någon annan redigerare) redigerar du csproj-filen för att lägga till paketet som ett **PackageReference**-element.  I det här exemplet används en specifik version av **Microsoft.Azure.Management.ResourceManager**, som är den bästa metoden, men [flytande versioner](/nuget/consume-packages/package-references-in-project-files#floating-versions) av NuGet stöds också.

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp1.1</TargetFramework>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.Azure.Management.ResourceManager" Version="1.4.0-preview" />
  </ItemGroup>

</Project>
```

> [!TIP]
> I föregående versioner av .NET Core användes project.json-filer istället för csproj.  Mer information om att mappa project.json till csproj [finns i det här dokumentet](/dotnet/articles/core/tools/project-json-to-csproj).
