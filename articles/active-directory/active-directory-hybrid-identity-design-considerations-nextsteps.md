#Überlegungen zum Entwurf der Azure Active Directory-Hybrid-Identität – Nächste Schritte
Nachdem Sie die Anforderungen definiert und alle Optionen für Ihre Lösung zur Verwaltung mobiler Geräte (Mobile Device Management, MDM) untersucht haben, können Sie die nächsten Schritte zur Bereitstellung der unterstützenden Infrastruktur ausführen, die für Sie und Ihr Unternehmen geeignet ist.

##Hybrid-Identitätslösungen
Die Nutzung spezieller Lösungsszenarien, die für Ihre Anforderungen geeignet sind, ist eine gute Möglichkeit, die Bereitstellung einer Infrastruktur für die Verwaltung mobiler Geräte zu prüfen und die damit verbundenen Details zu planen. Die folgenden Lösungen verdeutlichen die häufigsten Szenarien für die Verwaltung mobiler Geräte:

- Die [Verwalten von mobilen Geräten und PCs in Enterprise-Umgebungen Lösung](https://technet.microsoft.com/en-us/library/dn582037.aspx) hilft Ihnen, erweitern Ihre lokalen System Center 2012 Configuration Manager-Infrastruktur in die Cloud mit Microsoft Intune Verwaltung mobiler Geräte. Diese Hybrid-Infrastruktur dient IT-Spezialisten in mittleren und großen Unternehmen als Unterstützung bei der Einrichtung von BYOD und des Remotezugriffs, während gleichzeitig der Verwaltungsaufwand reduziert wird.
- Die [Verwalten mobiler Geräte für Configuration Manager 2007-Lösung](https://technet.microsoft.com/en-us/library/dn508400.aspx) hilft Ihnen die Verwaltung mobiler Geräte, wenn Ihre Infrastruktur über ein System Center Configuration Manager 2007 befindet. Bei dieser Lösung wird Ihnen gezeigt, wie Sie einen einzelnen Server mit System Center 2012 Configuration Manager einrichten, damit Sie dann Microsoft Intune ausführen und die Vorteile in Bezug auf die Verwaltung mobiler Geräte nutzen können, die diese Anwendung bietet.
- Die [Verwaltung mobiler Geräte in kleinen Umgebungen Lösung](https://technet.microsoft.com/en-us/library/dn715906.aspx) richtet sich für kleine Unternehmen, die zur Unterstützung von Mdm. Es wird erläutert, wie Sie Microsoft Intune zum Erweitern Ihrer aktuellen Infrastruktur verwenden, um die Verwaltung mobiler Geräte und BYOD zu unterstützen. Bei dieser Lösung wird das einfachste Szenario beschrieben, das für die Nutzung von Microsoft Intune in einer eigenständigen, nur für die Cloud bestimmten Konfiguration ohne lokale Server unterstützt wird.

## Dokumentation zur Hybrid-Identität
Informationen zur konzeptuellen und verfahrensorientierten Planung, Bereitstellung und Verwaltung sind hilfreich, wenn Sie Ihre Lösung für die Verwaltung mobiler Geräte implementieren:

- [Microsoft System Center](https://technet.microsoft.com/en-us/library/cc507089.aspx) Lösungen können Sie erfassen und die zusammengefassten Kenntnisse über Ihre Infrastruktur, Richtlinien, Prozesse und bewährten Methoden, damit Ihr IT-Personal verwaltbare Systeme einrichten und Vorgänge automatisieren können.
- [Microsoft Intune](https://technet.microsoft.com/en-us/library/jj676587.aspx) ist ein Verwaltungsdienst für Cloud-basierte Geräte, die Sie zum Verwalten von Computern und mobilen Geräten und die Daten Ihres Unternehmens schützen kann.
- [Verwaltung von Mobilgeräten für Office 365](https://technet.microsoft.com/en-us/library/ms.o365.cc.devicepolicy.aspx) ermöglicht es Ihnen, verwalten und schützen mobile Geräte, wenn sie Ihre Office 365-Organisation verbunden sind. Sie können MDM für Office 365 nutzen, um Sicherheitsrichtlinien und Zugriffsregeln für Geräte festzulegen und die Daten mobiler Geräte zu löschen, falls Sie verloren gehen oder gestohlen werden.
