# <a name="azure-cosmos-db-documentation-experiment"></a>Experiment med Azure DB Cosmos-dokumentation

Azure Cosmos-DB är en global och flera olika modeller databas. Eftersom du kan göra så mycket med Azure DB som Cosmos - från att skapa ett diagram för att skapa en databas för dokumentet eller migrera dina data från MongoDB, Cassandra eller Azure Table storage - ordna dokumentationen kan vara en utmaning. Vi vill så att din hjälp.

Vi vill testa organisationen för innehållsförteckningen Azure Cosmos DB och [landningssida](https://docs.microsoft.com/en-us/azure/cosmos-db/)som visas i följande bilder och vill vi gärna höra dina synpunkter på dessa föreslagna ändringar.

## <a name="to-load-both-views"></a>Att läsa in vyn

1. Om du vill öppna vyn sida vid sida, navigera till den [landningssida](https://docs.microsoft.com/en-us/azure/cosmos-db/) från valfri webbläsare. 
2. Bilder som ovan för att avgöra vilken version du ser, ursprungliga och experiment. 
3. Öppna en InPrivate- eller incognito webbläsarfönster och Läs in den [landningssida](https://docs.microsoft.com/en-us/azure/cosmos-db/) i nytt fönster i webbläsaren och använda igen avbildningar för att avgöra vilken version som du ser. 

   Om du inte har båda vyerna som finns, Stäng och öppna ett nytt InPrivate- eller incognito-fönster och Läs in den [landningssida](https://docs.microsoft.com/en-us/azure/cosmos-db/) igen. Det kan ta lite tid att hämta båda versionerna som lästs in i separata fönster. Uppdatera fönstret incognito kommer inte att läsa in den andra versionen, måste du öppna ett nytt fönster. 

4. Du kan sedan ändra windows för att korrigera sida vid sida eller växla mellan lägen ska jämföras. 

5. När du har haft möjlighet att bekanta dig med organisation och layout av experiment, klickar du på den **experimentera undersökning** knappen nedan för att ge feedback. Skicka formuläret senast midnatt 4 April 2018 så att feedbacken kan granskas och ingår.
