# <a name="billing-options"></a>Betalningsalternativ  

## <a name="commercial-considerations-on-the-marketplace"></a>Affärsmässiga överväganden på marketplace  
Marketplace delar inte intäkter för följande typer av listan.  
*   Visa lista  
*   Utvärdering  
*   Hantera med hjälp av faktureringsmodellen Bring Your Own License (BYOL)

Det går inte att debiteras ytterligare avgifter för att delta i butiker på marketplace.  
*   Mer information finns i Deltagandepolicyer för Azure Marketplace sidan finns på [azure.microsoft.com/support/legal/marketplace/participation-policies](https://azure.microsoft.com/support/legal/marketplace/participation-policies).  

## <a name="pay-as-you-go-and-bring-your-own-license-billing-options"></a>Betala per användning och ta med din egen licens betalningsalternativ  
När du använder en användningsbaserad fakturering modell som en publiceringsalternativ delade programvaran användningsbaserad licensiering intäkter är 80/20 mellan dig och Microsoft. Ett erbjudande kan skilja sig med hjälp av både betala per användning och BYOL faktureringsmodeller. Två faktureringsmodeller samexistera på nivån erbjudande som separata SKU: er. Du konfigurerar faktureringsmodellerna i ditt erbjudande i partnerportalen i molnet. 

Överväg följande exempel.  
*   Om du aktiverar alternativet betala per användning kan ha du följande resultat.  
    | Din licens kostnad | $1.00 per timme |  
    |:--- |:--- |  
    | Användningen av Azure-kostnad (D1/1-kärna) | $0.14 per timme |  
    | *Faktureras kunden av Microsoft* | *$1.14 per timme* |  

    Microsoft fakturerar $1.14 per timme för användningen av dina publicerad virtuell datoravbildning i det här scenariot.  
    | Microsoft fakturor | $1.14 per timme |  
    |:--- |:--- |  
    | Microsoft betalar du 80% av dina kostnader för licens | $0.80 per timme |  
    | Microsoft behåller 20% av dina kostnader för licens | $0.20 per timme |  
    | Microsoft behåller kostnaden för Azure-användning | $0.14 per timme |  

*   Om du aktiverar alternativet BYOL kan ha du följande resultat.  
    | Din licens kostnad | Licensavgiften förhandlas och faktureras av dig |  
    |:--- |:--- |  
    | Användningen av Azure-kostnad (D1/1-kärna) | $0.14 per timme |  
    | *Faktureras kunden av Microsoft* | *$0.14 per timme* |  

    Microsoft fakturerar $0.14 per timme för användningen av dina publicerad virtuell datoravbildning i det här scenariot.  
    | Microsoft fakturor | $0.14 per timme |  
    |:--- |:--- |  
    | Microsoft behåller kostnaden för Azure-användning | $0.14 per timme |  
    | Microsoft behåller 0% av dina kostnader för licens | $0,00 per timme |  

## <a name="single-billing-and-payment-methods"></a>Enskild fakturering och betalning  
En viktig fördel med att använda Transact lista alternativet är att licensieringskostnader och Azure-användning är enkel faktureras direkt till kunden.  

I scenario Microsoft fakturerar och samlar in för din räkning. Microsoft fakturering undanröjer behovet av att skapa din egen inköp relation med kunden. Enskild fakturering kan spara tid och resurser. Enskild fakturering kan hjälpa dig att fokusera på landing försäljningen i stället för att samla in fakturan. 

## <a name="enterprise-agreement"></a>Enterprise Agreement  
Om du är kund med Enterprise Agreement (EA) med Microsoft, kan du använda ditt EA för att betala för Microsoft-produkter. Du debiteras för produkter, inklusive din Azure-användning. Med hjälp av ditt EA betala är utformad för organisationer som vill licensiera programvara och molntjänster i minst tre år. Du sprida ut betalningar istället för en att göra förskottsbetalning. Om du använder betala per användning transact listtyp kan sedan faktureringen för dina programlicenskostnaderna följer kvartalsvis EA överförbrukning faktureringsperioden.  

### <a name="monetary-commitment"></a>Åtagandebelopp  
Om du är EA-kund kan du lägga till Azure i ditt avtal. Du lägger till Azure i ditt avtal genom att betala ett åtagandebelopp till Azure. Ditt betalningsåtagande förbrukas sedan under året. Ditt åtagande innehåller en kombination av användningen av det breda utbudet av tjänster i Azure.
