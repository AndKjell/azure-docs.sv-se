# <a name="billing-options"></a>Alternativ för fakturering  

## <a name="commercial-considerations-on-the-marketplace"></a>Affärsmässiga överväganden på marketplace  
Marketplace delar inte intäkter för följande typer av listan.  
*   Visa lista  
*   Utvärdering  
*   Transact med fakturering modellen Bring Your Own License (BYOL)

Du är inte fakturerade ytterligare avgifter för deltagande i skyltfönster på marketplace.  
*   Mer information finns i sidan finns på Azure Marketplace deltagande principer [azure.microsoft.com/support/legal/marketplace/participation-policies](https://azure.microsoft.com/support/legal/marketplace/participation-policies).  

## <a name="pay-as-you-go-and-bring-your-own-license-billing-options"></a>Betala per användning och sätta egna licensalternativ fakturering  
När du använder en modell med betalning per användning fakturering som ett publiceringsalternativ delade användningsbaserad programvaran licensiering intäkter är 80/20 mellan dig och Microsoft. Ett erbjudande kan sättas med både betala per användning och BYOL fakturering modeller. Fakturering modellerna samexistera på nivån erbjudande som separata SKU: er. Du kan konfigurera fakturering modeller i erbjudandet på partnerportalen för molnet. 

Tänk på följande exempel.  
*   Om du aktiverar alternativet betala per användning, har du följande resultat.  
    | Din licens kostnad | 1,00 $ per timme |  
    |:--- |:--- |  
    | Azure användning kostnaden (D1/1 kärna) | $0.14 per timme |  
    | *Kunden faktureras av Microsoft* | *$1.14 per timme* |  

    Microsoft debiterar $1.14 per timme för användning av en publicerad virtuell datoravbildning i det här scenariot.  
    | Microsoft växlar | $1.14 per timme |  
    |:--- |:--- |  
    | Microsoft betalar 80% av licens-kostnad | $0,80 per timme |  
    | Microsoft håller 20% av licens-kostnaderna | $0,20 per timme |  
    | Microsoft håller kostnaden Azure användning | $0.14 per timme |  

*   Om du aktiverar alternativet BYOL har du följande resultat.  
    | Din licens kostnad | Licens avgift förhandlades och fakturerats av du |  
    |:--- |:--- |  
    | Azure användning kostnaden (D1/1 kärna) | $0.14 per timme |  
    | *Kunden faktureras av Microsoft* | *$0.14 per timme* |  

    Microsoft debiterar $0.14 per timme för användning av en publicerad virtuell datoravbildning i det här scenariot.  
    | Microsoft växlar | $0.14 per timme |  
    |:--- |:--- |  
    | Microsoft håller kostnaden Azure användning | $0.14 per timme |  
    | Microsoft håller 0% av licens-kostnad | 0.00 $ per timme |  

## <a name="single-billing-and-payment-methods"></a>Enkel fakturering och betalning  
En viktig fördel med att använda Transact med alternativet är att din licensieringskostnader och Azure användning enskild debiteras direkt till kunden.  

I scenario Microsoft debiterar och samlar in för din räkning. Microsoft fakturering undanröjer behovet av du vill skapa egna inköp relation med kunden. Enkel fakturering kan spara tid och resurser. Enkel fakturering kan hjälpa dig att fokusera på hamnar försäljning, i stället för att samla in växeln. 

## <a name="enterprise-agreement"></a>Enterprise Agreement  
Om du är en kund med Enterprise-avtal (EA) med Microsoft kan du använda din EA betala för Microsoft-produkter. Du kan debiterar produkter, inklusive din användning av Azure. Med hjälp av din EA betala är utformad för organisationer som vill licens programvara och molntjänster i minst tre år. Du sprida ut betalningar i stället för en direkta betalning. Om du använder betala per användning transact listtyp, faktureringen för programvaran licensieringskostnader följer kvartalsvis EA överförbrukning faktureringsperioden.  

### <a name="monetary-commitment"></a>Åtagandebelopp  
Om du är en EA-kund kan du lägga till Azure till ditt avtal. Du lägger till Azure ditt avtal genom att göra en summa i förskott till Azure. Din summa förbrukas under året. Ditt engagemang innehåller en kombination av användning av flera olika tjänster i Azure.
